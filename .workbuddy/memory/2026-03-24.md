**1. 保存持仓失败**

- 修复：`API = ''`（空字符串，相对路径），`fetchData()` catch 分支不再覆盖已有真实数据

**2. 添加持仓/记录开仓按钮时有时无**
- 根因：`switchManageTab('settings')` 时执行 `addBtn.style.display = 'none'`，但切回 stock/options tab 时没有恢复 `style.display`
- 修复：stock 和 options 分支先 `addBtn.style.display = ''` 再设置颜色和 onclick

**3. 持仓明细与投资总览数据一致性**
- 修复：持仓明细 Summary Bar 从 3 卡扩展为 4 卡，新增"总资产（含现金）"和"现金储备"，与投资总览 KPI 完全对应

- `renderOverview()` KPI 副标题动态更新（涨跌幅、现金占比百分比）

**4. 期权在全站展示**
- 新增 `loadOptionsOverview()` 函数：异步调用 `/api/options/positions` 和 `/api/options/stats`
- **投资总览**：新增"期权持仓摘要"行（有期权时显示，无期权隐藏）：4个KPI卡 + 当前持仓中最多5条速览

## 2026-03-24 20:55 管理后台+注册+期权展示修复（v3.3）

### 修复了4个新问题

**1. 注册/管理员添加用户失败（Internal Server Error）**
- 根因：`register()` 函数在 `with get_db()` 块内部调用了 `log_action()`，而 `log_action()` 自己又 `with get_db()` 打开了新连接，SQLite 发生 `database is locked`
- 修复：`log_action()` 新增可选 `db` 参数，有传则复用，没有则自开连接；`register()` 内改为 `log_action(uid, "register", ..., db=db)`
- 同时给 `get_db()` 添加 `timeout=15` 和 `PRAGMA busy_timeout=10000`，彻底防止并发锁死

**2. 三个文件 API 硬编码 localhost**
- `claw_admin.html` 和 `claw_login.html` 的 `const API = 'http://localhost:8000'` 改为 `''`（相对路径）
- 现在登录/注册/管理员在手机或局域网 IP 访问时均可正常工作

**3. Mike 字样清除**
- 3处全部替换：「基于 Mike 体系」→「专属策略体系」，「Mike 策略原则」→「Sell Put 核心原则」，「参考 Mike 体系」→「根据投资框架分析」

## 2026-03-24 21:20 正股持仓显示MOCK数据修复（v3.4）

### 根因分析
- `fetchData()` 首次失败时，`dashData = MOCK_DATA`（含假持仓 MSFT/TSLA/META），然后 `renderManage()` 显示假数据
- 60秒自动刷新失败时会 toast"数据刷新失败"但显示的依然是 MOCK 假数据
- 后端 `/api/dashboard` 完全正常，是前端在首次 fetch 失败时的 fallback 逻辑有问题

### 修复内容
1. **fetchData 重试机制**：首次失败时不再立即显示 MOCK_DATA，而是 3 秒后自动重试
2. **_isMockData 标志**：区分真实数据和占位数据，避免假数据污染后续渲染
3. **manage 页切换优化**：切换到持仓管理页时，如果是 mock 数据立即重新 fetch，否则 re-render 最新状态
4. **renderManage 加载提示**：如果是占位数据，表格显示"数据加载中，请稍候…"旋转图标，而不是假持仓
5. **服务器已重启**，端口 8000，确认 Z 账户有 9 条正股持仓
