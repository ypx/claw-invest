# CLAW Dashboard 版本管理

## 版本命名规则

`v{主版本}.{次版本}.{修订号}_{YYYYMMDD}`

- 主版本：重大架构变更或功能重构
- 次版本：新增功能模块
- 修订号：bug 修复或 UI 微调

---

## 版本历史

### v1.4.0 (2026-03-31) ✅ 当前稳定版
**备份时间**: 2026-03-31 08:35

**修复内容**:
- **SPA路由刷新问题**
  - 修复页面刷新后总是跳转到"投资总览"的问题
  - 根本原因：`getHashPage()` 函数引用了未定义的 `PAGE_TITLES` 变量
  - 修复：改为直接检查 DOM 元素 `document.getElementById('page-' + hash)` 是否存在
  - 影响：所有页面刷新后都能正确保持在当前页面

---

### v1.3.0 (2025-03-25)
**备份时间**: 2026-03-25 10:42

**修复内容**:
- **修复手机端整体向右偏移**
  - 根本原因：`#main { margin-left: 240px; }` 在手机端未重置
  - 修复：`#main { margin-left: 0; width: 100%; max-width: 100vw; }`
  - 添加 `body { overflow-x: hidden; }` 防止整体溢出

- **优化KPI布局（数据少时并排）**
  - 投资总览、持仓明细、TSLA、市场行情等页面的KPI统一为2x2网格
  - 480px以下仍保持2x2，缩小间距和padding
  - 图标尺寸统一缩小为36x36px

- **修复内联grid样式溢出**
  - 覆盖所有 `[style*="grid-template-columns:1fr 1fr"]` 为单列
  - 覆盖所有 `[style*="auto-fill"]` 为单列

## 回滚操作

如需回滚到某个版本：

```bash
# 回滚到 v1.0.0
cp versions/dashboard_v1.0.0_20250325.html frontend/claw_dashboard_enhanced.html
cp versions/login_v1.0.0_20250325.html frontend/claw_login.html
```

## 开发流程建议

1. **修改前**：先备份当前版本到 versions/
2. **修改中**：在 frontend/ 文件上直接修改
3. **测试后**：确认无误后，更新版本号并再次备份
4. **如需回滚**：从 versions/ 复制对应版本覆盖
