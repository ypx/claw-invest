📈 CLAW 投资策略模块 - 产品需求文档

版本： v1.0 | 日期： 2026-03-26 | 状态： 规划中

1. 产品概述

目标： 为用户提供个性化的投资策略推荐，基于用户画像（资金规模、风险偏好、投资目标、经验水平）智能匹配适合的投资策略组合。

定位： 现有期权策略的升级版，从单一期权策略扩展为完整的投资策略体系，包含保守型、稳健型、平衡型、进取型等多种策略。

2. 核心概念

2.1 用户画像 (UserProfile)

维度选项说明
capital_scale小资金 / 中等 / 大资金5万以下 / 5-50万 / 50万+
risk_tolerance保守 / 稳健 / 平衡 / 激进可接受的最大亏损程度
investment_goal保值 / 增值 / 增长 / 高回报投资目标
experience_level小白 / 有经验 / 专业投资经验水平
time_horizon短期 / 中期 / 长期投资周期

2.2 投资策略 (Strategy)

维度说明
id策略唯一标识
name策略名称
description策略简要说明
risk_level风险等级 1-5（1=极保守，5=激进）
min_capital建议最低资金门槛
expected_return预期收益范围（如 8-12%）
suitable_profiles匹配的画像标签组合
allocations资产配置建议列表
tags策略标签（ETF、期权、加密等）

3. 功能清单

3.1 新功能模块

功能优先级说明
风险测评引导P0新用户引导完成5道题问卷，生成用户画像
画像管理P0查看/修改个人画像，重新测评
个性化推荐P0基于用户画像推荐最匹配的策略（匹配度展示）
策略广场P1全部策略列表，支持按风险/资金/类型筛选
策略详情页P1配置建议、标的示例、策略说明、订阅/自选
策略订阅P2订阅策略后，相关标的进入自选监控
画像修改入口P2个人中心增加"修改投资画像"入口

3.2 与现有系统整合

整合点说明
导航栏入口新增"投资策略"入口（独立于期权策略）
个人中心投资偏好页增加画像维护功能
持仓管理持仓可关联策略标签
数据打通复用现有行情数据、持仓数据

5. 数据库设计

5.1 新增表：user_profiles

id: INTEGER PRIMARY KEY
user_id: INTEGER (外键)
capital_scale: TEXT (小资金/中等/大资金)
risk_tolerance: TEXT (保守/稳健/平衡/激进)
investment_goal: TEXT (保值/增值/增长/高回报)
experience_level: TEXT (小白/有经验/专业)
time_horizon: TEXT (短期/中期/长期)
is_completed: BOOLEAN (是否完成测评)
created_at: TIMESTAMP
updated_at: TIMESTAMP

5.2 新增表：strategies (预置策略)

id: INTEGER PRIMARY KEY
name: TEXT (策略名称)
description: TEXT (策略说明)
risk_level: INTEGER (1-5)
min_capital: TEXT (资金门槛描述)
expected_return: TEXT (收益范围)
suitable_profiles: TEXT (JSON数组，匹配的画像)
allocations: TEXT (JSON数组，配置建议)
tags: TEXT (JSON数组，策略标签)
is_active: BOOLEAN (是否启用)

5.3 新增表：user_strategy_subscriptions

id: INTEGER PRIMARY KEY
user_id: INTEGER
strategy_id: INTEGER (外键)
subscribed_at: TIMESTAMP

6. API 设计

接口方法说明
/api/strategy/profileGET获取当前用户画像
/api/strategy/profilePOST提交问卷答案，生成/更新画像
/api/strategy/recommendGET获取个性化推荐（基于画像）
/api/strategy/listGET获取全部策略（支持筛选）
/api/strategy/{id}GET获取策略详情
/api/strategy/{id}/subscribePOST订阅策略
/api/strategy/subscriptionsGET获取用户订阅的策略

9. 开发计划

阶段一：基础功能

  • 数据库建表（user_profiles, strategies, subscriptions）
  • 策略数据初始化（预置12-15个策略）
  • 后端 API 开发

阶段二：前端页面

  • 投资策略首页（推荐区 + 策略广场）
  • 策略详情页
  • 问卷引导页（5道题）

阶段三：整合与优化

  • 导航栏入口整合
  • 个人中心画像入口
  • 策略订阅功能
  • 持仓关联策略标签
备注： 本需求文档基于用户讨论结果生成，期权策略模块保持不变，作为策略广场中的一个分类存在。
