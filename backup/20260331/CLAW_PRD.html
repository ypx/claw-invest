① 产品概述

目标用户

主用户：Z（也可开放给更多个人投资者）

  • 个人美股投资者，保守型，长期持有 AI / 科技 / 太空方向正股
  • 保留大量现金仓位（约 59%），偶尔做 Sell Put 期权（尤其是 TSLA）
  • 使用老虎证券作为主要券商
  • 无代码经验，会 Excel 公式，需要 直观、中文、手机可用 的界面
  • 参考 Mike是麥克（YouTube @mike1111）的投资逻辑框架作决策辅助

核心价值主张

一屏掌握
全局持仓状态

主动提醒
风险自动预警
现金超目标、集中度过高、VIX 异常，智能触发通知
期权辅助
Sell Put 策略
推荐高概率 Put 合约，追踪期权收入历史
随时随地
手机/电脑通用

③ 功能缺口分析（待实现）

⚠️
以下功能是让系统从"可用"升级为"好用"的关键。优先级分为 P0（必做）、P1（重要）、P2（锦上添花）。
优先级 功能名称 描述 影响 难度
P0 实时股价接入 对接 Alpha Vantage API，自动更新每只持仓的 current_price 核心数据准确性 中
P0 净值历史曲线 每日存储一次快照，展示资产净值走势图（月/季/年） 直观看懂投资表现 中
P0 期权持仓管理 可添加/追踪 Sell Put 合约（行权价、到期日、权利金、开仓价格） 核心业务场景 中
P1 加密资产追踪 支持 BTC/ETH 持仓录入，接入 CoinGecko 实时价格 覆盖完整资产 低
P1 定投计划管理 记录每月定投计划（标的 + 金额），追踪执行情况 辅助纪律执行 低
P1 移动端推送通知 重要风险警报（VIX 异常、持仓大跌）推送到手机 及时响应市场 高
P1 交易记录日志 手动记录买入/卖出操作，自动更新持仓成本和数量 成本基础准确 中
P1 密码修改 + 个人设置 用户可自行修改密码、昵称、显示货币、时区等 基础账户体验 低
P1 数据导出（CSV/PDF） 导出持仓明细、交易历史，方便税务申报和备份 实用性 低
P2 AI 投资顾问对话 接入 LLM，用户可问"TSLA 现在适合 Sell Put 吗"，基于持仓上下文回答 差异化竞争力 高
P2 收益率对比基准 将组合表现与 SPY / QQQ 对比，直观看是否跑赢大盘 投资成效评估 中
P2 股息追踪 记录持仓股的股息发放记录和预期年化股息收益 收益完整性 低
P2 分享报告（只读链接） 生成可分享的持仓报告链接（不含成本价，保护隐私） 社交分享 中
P2 黑暗/明亮主题切换 当前仅有暗色模式，添加亮色模式 使用舒适度 低
P2 PWA 离线支持 添加 Service Worker，离线可查看上次缓存的数据 移动端体验 中
P2 多账户聚合 支持老虎、富途等多个券商账户独立管理后汇总 未来扩展性 高
P2 目标持仓规划 设置理想仓位比例，自动计算距离目标差多少钱 辅助决策 低
P2 新闻实时推送 对接 Finnhub / NewsAPI，自动推送持仓相关新闻 信息聚合 中

⑦ 完整上线方案

🎯
目标：随时随地通过手机/电脑访问，无需开着本地电脑。以下是三种方案，按推荐程度排序。

方案 A：云服务器部署（推荐）

原理：租一台境外 VPS，把 Claw 跑在上面，绑定域名，手机/电脑通过 HTTPS 访问。

推荐配置

  • 服务器：腾讯云 / 阿里云轻量应用服务器，2核 2GB，约 ¥50-100/月
  • 系统：Ubuntu 22.04 LTS
  • 域名：可注册一个 claw.投资.com，或直接用 IP
  • HTTPS：Let's Encrypt 免费证书，Nginx 反向代理

部署步骤概述

  1. SSH 登录服务器，安装 Python 3.11 + Docker
  2. 上传代码，修改 CLAW_SECRET、DB_PATH 环境变量
  3. 将数据库从 SQLite 迁移至 PostgreSQL（Phase 2）
  4. 配置 Nginx 反代，申请 SSL 证书
  5. 用 systemd 或 PM2 保持进程持久运行
💡
域名 + 服务器 + SSL 全部搞定后，访问地址就是 https://claw.你的域名.com，手机加书签就像 App 一样。

方案 B：Railway / Render 免费托管

适合测试阶段，免费，无需运维，但有一定限制（如冷启动延迟）。

  • Railway.app：直接从 GitHub 部署 FastAPI，自动处理 HTTPS，有免费额度
  • Render.com：类似 Railway，免费版有休眠机制（15 分钟无请求后关闭）
  • 数据库：可使用 Railway 提供的 PostgreSQL 插件
  • 缺点：免费版冷启动慢（约 30 秒），不适合生产使用

方案 C：本地 + 内网穿透（应急）

如果不想花钱，可用 Cloudflare Tunnel 或 ngrok 将本地服务暴露到公网。

  • Cloudflare Tunnel：完全免费，稳定，有公网域名，推荐
  • 缺点：依赖本地 Mac 必须开机，不适合长期使用

生产环境安全清单

检查项当前状态处理方式
SECRET_KEY 保密 危险（硬编码） 改为 os.getenv("CLAW_SECRET") 并在服务器设置环境变量
HTTPS 加密 本地没有 部署时用 Nginx + Let's Encrypt，所有流量走 HTTPS
数据库备份 无 每天定时备份 SQLite 文件或 PostgreSQL dump 到云存储
速率限制 无 添加 SlowAPI 限流，防止暴力破解登录接口
CORS 白名单 当前 * 生产环境改为只允许你的域名
日志监控 仅控制台 配置文件日志 + 错误邮件告警（或接入 Sentry）
